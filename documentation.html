1. Loading Files

CorpuScript offers flexible options for loading your text data, whether you’re working with individual files or entire directories. This section guides you through the process of importing your .txt files into the application.

1.1. Loading Individual Files

  1. Access the Open Files Dialog:
    • Via Menu: Click on File > Open Files in the menu bar.
    • Via Toolbar: Click the "Open Files" icon in the toolbar.
  2. Select Files:
    • In the dialog that appears, navigate to the location of your .txt files.
    • Select one or more .txt files by holding the Ctrl key (or Cmd on Mac) and clicking on each file.
  3. Add to File List:
    • Click "Open".
    • The selected files will now appear in the "Selected Files" list on the left panel of the main window.

1.2. Loading an Entire Directory

  1. Access the Open Directory Dialog:
    • Via Menu: Click on File > Open Directory.
    • Via Toolbar: Click the "Open Directory" icon in the toolbar.
  2. Select Directory:
    • In the dialog that appears, navigate to the directory containing your .txt files.
  3. Add to File List:
    • Click "Select Folder".
    • All .txt files within the chosen directory and its subdirectories will be added to the "Selected Files" list.

Note: CorpuScript supports batch processing, allowing you to handle large volumes of text files efficiently.

2. Configuring Preprocessing Options

CorpuScript provides a robust set of customizable preprocessing options, allowing you to tailor the cleaning process to meet the specific requirements of your project. These options are accessible through the Processing Parameters dialog, which is divided into two main sections: General and Advanced.

2.1. Accessing the Processing Parameters Dialog

You can access the Processing Parameters dialog through two convenient methods:

  1. Via Menu:
    • Navigate to Settings > Processing Parameters in the menu bar.
  2. Via Toolbar:
    • Click the gear icon located in the toolbar for quick access.

2.2. General Tab

The General tab contains a series of checkboxes that enable or disable specific preprocessing tasks. Each option serves a distinct purpose in cleaning and preparing your text data.

2.2.1. Remove Line Breaks

  • Description: Eliminates all newline characters (\n) from the text, converting multi-line text into a single continuous line.
  • Use Case: Ideal for preparing text data that should not contain any line breaks, such as when analyzing continuous narratives or preparing data for models that require uninterrupted text streams.
  • How to Use: Simply check the box labeled "Remove Line Breaks". The preprocessing pipeline will automatically remove all line breaks during processing.

2.2.2. Lowercase Conversion

  • Description: Transforms all characters in the text to lowercase, ensuring uniformity in case-sensitive analyses.
  • Use Case: Essential for tasks like tokenization, frequency analysis, and other NLP processes where case distinctions are irrelevant or may introduce inconsistencies.
  • How to Use: Check the box labeled "Lowercase Conversion". All text will be converted to lowercase before further processing.

2.2.3. Whitespace Normalization

  • Description: Standardizes whitespace by removing redundant spaces, tabs, and other whitespace characters, ensuring consistent spacing throughout the text.
  • Use Case: Prevents issues related to inconsistent spacing, which can affect the accuracy of text analysis and processing algorithms.
  • How to Use: Enable this option by checking the "Whitespace Normalization" box. The pipeline will clean up excess whitespace automatically.

2.2.4. Stopword Removal

  • Description: Removes common, non-informative words (stopwords) such as "the", "and", "is", etc., from the text.
  • Use Case: Reduces noise in text data, enhancing the performance of algorithms by focusing on meaningful words that contribute more significantly to the analysis.
  • How to Use: Check the box labeled "Stopword Removal". You can further customize the list of stopwords in the Advanced tab if needed.

2.2.5. Strip HTML Tags

  • Description: Removes all HTML tags from the text, extracting plain text content from web-scraped or marked-up documents.
  • Use Case: Crucial when dealing with text data sourced from websites or any documents containing HTML markup, ensuring only the textual content is processed.
  • How to Use: Enable this option by checking the "Strip HTML Tags" box. All HTML tags will be stripped during preprocessing.

2.2.6. Remove Diacritics

  • Description: Strips away diacritical marks (accents) from characters, converting them to their base forms (e.g., "é" to "e").
  • Use Case: Useful for simplifying text for analysis, especially in languages with diacritics, to ensure uniformity and prevent mismatches in text processing tasks.
  • How to Use: Check the box labeled "Remove Diacritics". Characters with diacritics will be converted to their non-diacritic forms during preprocessing.

2.2.7. Remove Greek Letters

  • Description: Filters out Greek script characters from the text.
  • Use Case: Necessary when processing text data that should exclude Greek characters, possibly to focus on specific scripts or languages.
  • How to Use: Enable this option by checking the "Remove Greek Letters" box. All Greek letters will be removed during preprocessing.

2.2.8. Remove Cyrillic Script

  • Description: Filters out Cyrillic script characters from the text.
  • Use Case: Similar to removing Greek letters, this is useful when the corpus should exclude Cyrillic script, focusing on other scripts or languages.
  • How to Use: Check the box labeled "Remove Cyrillic Script" to enable the removal of Cyrillic characters during preprocessing.

2.2.9. Remove Superscript and Subscript Characters

  • Description: Removes superscript and subscript characters, which are typically used for annotations, mathematical expressions, or specialized formatting.
  • Use Case: Cleans text data by removing characters that may not be relevant to linguistic analysis or could interfere with text processing algorithms.
  • How to Use: Enable this option by checking the "Remove Superscript and Subscript Characters" box. These characters will be filtered out during preprocessing.

2.2.10. Normalize Unicode

  • Description: Normalizes the text to a standard Unicode form (NFKC), ensuring consistency in character representation across the dataset.
  • Use Case: Essential for maintaining data integrity, especially when dealing with text from multiple sources or languages, preventing discrepancies caused by different Unicode representations.
  • How to Use: Check the box labeled "Normalize Unicode". The text will be normalized to NFKC form during preprocessing.

2.2.11. Lemmatization

  • Description: Reduces words to their base or dictionary form (lemmas), aiding in linguistic analysis by grouping together different forms of a word.
  • Use Case: Improves the accuracy of analyses such as frequency counts, sentiment analysis, and topic modeling by treating different grammatical forms of a word as a single entity.
  • How to Use: Check the box labeled "Lemmatization". The preprocessing pipeline will automatically lemmatize all words during processing.

2.2.12. Sentence Tokenization

  • Description: Splits the text into individual sentences, facilitating sentence-level analyses and processing.
  • Use Case: Essential for tasks such as sentiment analysis, syntactic parsing, and any application requiring sentence boundaries.
  • How to Use: Enable this option by checking the "Sentence Tokenization" box. The text will be divided into sentences during preprocessing.

2.2.13. Word Tokenization

  • Description: Divides the text into individual words or tokens, essential for word-level analysis and processing.
  • Use Case: Fundamental for most Natural Language Processing (NLP) tasks, including frequency analysis, machine learning models, and more.
  • How to Use: Check the box labeled "Word Tokenization". The text will be split into words during preprocessing.

2.2.14. Remove Bibliographical References

  • Description: Automatically identifies and removes in-text bibliographical references (e.g., citations like (Smith, 2020)), cleaning up the text for analysis.
  • Use Case: Essential for academic texts and research papers where citations can interfere with textual analysis by introducing non-content elements.
  • How to Use: Enable this option by checking the "Remove Bibliographical References" box. The preprocessing pipeline will remove all bibliographical references during processing.

2.3. Advanced Tab

The Advanced tab offers more granular control over the preprocessing process, allowing users to define custom patterns and specify additional characters to remove. This is particularly useful for handling specialized text cleaning requirements that go beyond the general options provided.

2.3.1. Custom Regex Filtering

  • Description: Allows users to define custom regular expressions (regex) to perform advanced text filtering and extraction based on specific patterns.
  • Use Case: Enables complex text manipulation tasks, such as extracting specific patterns, removing certain phrases, or any task that requires pattern-based processing not covered by the general options.
  • How to Use:
    1. Open the Advanced Pattern Builder:
      • Click the "Set Pattern" button within the Advanced tab. This opens the Advanced Pattern Builder wizard.
    2. Define Your Patterns:
      • Add a New Pattern:
        • Click the "Add Pattern" button to create a new pattern entry.
      • Specify Conditions:
        • Start Condition: Define the starting point of the pattern you want to match.
        • End Condition Type: Choose how the pattern should end. Options include:
          • Single Number: Ends after a single numeric digit.
          • Multiple Numbers: Ends after a specified number of numeric digits.
          • Specific Word: Ends when a particular word is encountered.
        • End Condition: Specify the exact end condition based on the selected type.
        • Number Length: If using Multiple Numbers, define the exact number of digits.
      • Configure Additional Settings:
        • Case Sensitivity: Choose whether the pattern matching should be case-sensitive.
        • Whole Word Matching: Decide if the pattern should match whole words only.
    3. Test Your Pattern:
      • Enter sample text in the Test Input section to see how your pattern matches and affects the text.
      • Adjust the pattern as necessary based on the test results.
    4. Save the Pattern:
      • Once satisfied, save the pattern. It will be applied during preprocessing.
Example:
  • Objective: Remove all URLs from the text.
  • Pattern Definition:
    • Start Condition: http
    • End Condition Type: Specific Word
    • End Condition: Space character (\s) or end of string
  • Outcome: This pattern will match and remove any URL starting with http and ending before a space or the end of the text.

2.3.2. Select Characters to Remove

  • Description: Provides a dialog for selecting specific characters or sequences of characters to remove from the text, offering precise control over unwanted symbols or patterns.
  • Use Case: Useful for eliminating specific symbols, emojis, or any other characters that are not handled by other preprocessing options, ensuring that only relevant text data remains.
  • How to Use:
    1. Open the Character Selection Dialog:
      • Click the "Select Characters to Remove" button within the Advanced tab. This opens the Character Selection dialog.
    2. Add Characters or Sequences:
      • Enter Characters: Type the characters or sequences you wish to remove in the input field.
      • Include Characters: Click the "Include" button to add them to the removal list.
      • Example: To remove emojis, you might enter specific emoji characters or patterns.
    3. Review Selected Items:
      • The selected characters or sequences will appear in the "Items to remove" list below.
    4. Remove Unwanted Selections:
      • To delete any selected character or sequence from the removal list, select it in the list and click the "Delete Selected" button.
    5. Finalize Selections:
      • Once all desired characters or sequences are listed, click "OK" to apply the changes. These characters will be removed during preprocessing.
Example:
  • Objective: Remove all numerical digits and specific symbols like # and $.
  • Steps:
    1. Enter 0-9, #, and $ in the input field.
    2. Click "Include" after each entry.
    3. Verify that all selected items appear in the list.
    4. Click "OK" to apply the removals.

2.4. Applying the Preprocessing Parameters

After configuring your desired preprocessing options in both the General and Advanced tabs:

  1. Confirm Settings:
    • Review all enabled options to ensure they align with your preprocessing goals.
  2. Apply Parameters:
    • Click the "OK" button at the bottom of the Processing Parameters dialog to save and apply your settings.
  3. Start Processing:
    • With the parameters set, proceed to load your text files and initiate the preprocessing workflow by clicking the "Process Files" button in the toolbar.

Note: It's recommended to experiment with different preprocessing configurations on a small subset of your data to observe their effects before processing the entire corpus. This approach helps in fine-tuning the settings for optimal results.

2.5. Example: Configuring Preprocessing for a Specific Task

Scenario: Preparing a corpus for sentiment analysis by removing URLs, converting text to lowercase, stripping HTML tags, and removing stopwords.

  1. Open Processing Parameters:
    • Click Settings > Processing Parameters or the gear icon in the toolbar.
  2. General Tab Configurations:
    • Enable Lowercase Conversion:
      • Check the "Lowercase Conversion" box to ensure all text is in lowercase.
    • Enable Remove Line Breaks:
      • Check the "Remove Line Breaks" box to merge multi-line text into a single line.
    • Enable Strip HTML Tags:
      • Check the "Strip HTML Tags" box to remove any HTML markup.
    • Enable Stopword Removal:
      • Check the "Stopword Removal" box to eliminate common, non-informative words.
  3. Advanced Tab Configurations:
    • Define Custom Regex Pattern to Remove URLs:
      1. Click "Set Pattern" to open the Advanced Pattern Builder.
      2. Add Pattern:
        • Start Condition: http
        • End Condition Type: Specific Word
        • End Condition: Space character (\s) or end of string.
      3. Test Pattern:
        • Enter sample text containing URLs to verify the pattern correctly identifies and removes them.
      4. Save Pattern:
        • Once satisfied, save the pattern to apply it during preprocessing.
  4. Apply Parameters:
    • Click "OK" to save and apply all settings.
  5. Process Files:
    • Load your text files using File > Open Files or File > Open Directory.
    • Click the "Process Files" button in the toolbar to start preprocessing.
    • Monitor the progress through the progress bar and status updates.
  6. Review Results:
    • After processing, review the cleaned text in the "Processed Text" tab.
    • Ensure that URLs have been removed, text is in lowercase, HTML tags are stripped, and stopwords are eliminated.

2.6. Best Practices for Configuring Preprocessing Parameters

  1. Understand Your Data:
    • Before configuring preprocessing options, analyze your text data to identify common patterns, unwanted characters, and specific cleaning requirements.
  2. Start Simple:
    • Begin with basic preprocessing steps like lowercase conversion and whitespace normalization to establish a clean foundation.
  3. Incrementally Add Advanced Options:
    • Gradually introduce advanced preprocessing options such as custom regex filtering and character removals as needed, ensuring each step positively impacts your data quality.
  4. Test Configurations:
    • Apply different preprocessing configurations on a small subset of your data to observe their effects and adjust settings accordingly.
  5. Document Your Settings:
    • Keep a record of the preprocessing parameters used for each project to ensure reproducibility and facilitate future adjustments.
  6. Leverage Advanced Features:
    • Utilize the Advanced tab for specialized cleaning tasks that address unique aspects of your text data, enhancing the overall quality and relevance of your corpus.

3. Processing Files

Once you've loaded your text files and configured the preprocessing parameters, you're ready to process your corpus. This section outlines the steps to apply the selected preprocessing options to your files.

3.1. Initiating the Processing Workflow

  1. Start Processing:
    • Click the "Process Files" button located in the toolbar or select File > Process Files from the menu.
  2. Monitor Progress:
    • A Processing Files dialog will appear, displaying a progress bar and status updates.
    • The progress bar indicates the completion percentage of the processing task.
    • The status label provides real-time information about the current file being processed and estimated time remaining.
  3. Handling Errors:
    • If any errors occur during processing (e.g., file read/write issues), they will be displayed in the status bar and logged for your reference.
    • You can choose to cancel the processing at any time by clicking the "Cancel" button in the dialog.

3.2. Concurrent Processing

  • Multithreading Support:
    • CorpuScript utilizes multithreading to process multiple files simultaneously, leveraging your system's CPU cores for optimal performance.
    • This ensures efficient handling of large corpora, reducing the total processing time.
  • Resource Management:
    • The application automatically manages thread pools to prevent system overload, ensuring smooth operation even with extensive datasets.

4. Viewing and Saving Results

After processing your files, CorpuScript provides options to view the original and processed texts, as well as save the results for future use.

4.1. Viewing Results

  1. Accessing the Text Tabs:
    • Navigate to the "Original Text" and "Processed Text" tabs located in the main window.
    • Original Text Tab:
      • Displays the content of the original, unprocessed text files.
    • Processed Text Tab:
      • Shows the text after preprocessing has been applied.
  2. Navigating Between Tabs:
    • Click on the respective tabs to switch between viewing the original and processed versions of your files.
  3. Search Functionality:
    • Use the search bar below the text tabs to locate specific terms or phrases within the text.
    • Utilize the "Previous" and "Next" buttons to navigate through search results.

4.2. Saving Results

  1. Saving Processed Files:
    • Click on File > Save Files or use the "Save Files" button in the toolbar.
  2. Choose Save Options:
    • Overwrite Original Files:
      • Select this option to replace the original .txt files with their processed versions.
      • Caution: This action is irreversible. Ensure you have backups if needed.
    • Save to a Different Directory:
      • Choose this option to save the processed files in a separate location, preserving the original files.
  3. Confirm Save Operation:
    • A confirmation dialog will appear, asking you to confirm your save preferences.
    • Review your selection and click "Yes" to proceed.
  4. Handling Save Errors:
    • If any issues arise during the save process (e.g., insufficient permissions), CorpuScript will notify you via a warning message.
    • Review the error details, address the underlying issue, and attempt to save again if necessary.

5. Troubleshooting

Encountering issues while using CorpuScript? This section provides solutions to common problems and tips to ensure smooth operation.

5.1. Error Loading Files

  • Symptom: Files fail to load or appear in the "Selected Files" list.
  • Possible Causes:
    • Unsupported file format (only .txt files are supported).
    • Corrupted or unreadable files.
  • Solutions:
    • Ensure that only .txt files are being loaded.
    • Verify the integrity of the files by opening them in a text editor.
    • Re-download or recover corrupted files if necessary.

5.2. Incorrect Regex Patterns

  • Symptom: Preprocessing does not remove or alter text as expected when using custom regex patterns.
  • Possible Causes:
    • Syntax errors in the regex pattern.
    • Misconfigured start and end conditions.
  • Solutions:
    • Double-check the regex syntax using online regex testers.
    • Ensure that start and end conditions are correctly defined in the Advanced Pattern Builder.
    • Refer to the Regex Documentation for guidance.

5.3. Processing Interruptions

  • Symptom: Processing stops unexpectedly or is too slow.
  • Possible Causes:
    • Insufficient system resources (CPU or memory constraints).
    • Extremely large files causing delays.
  • Solutions:
    • Close other applications to free up system resources.
    • Break down large corpora into smaller batches and process them sequentially.
    • Monitor system performance to identify bottlenecks.

5.4. Save Operation Failures

  • Symptom: Unable to save processed files or overwrite originals.
  • Possible Causes:
    • Lack of write permissions in the target directory.
    • Files are open in another application, preventing overwriting.
  • Solutions:
    • Ensure you have the necessary permissions to write to the target directory.
    • Close any applications that might be accessing the files.
    • Choose an alternative directory to save the processed files.

5.5. Application Crashes or Freezes

  • Symptom: CorpuScript becomes unresponsive or crashes during operation.
  • Possible Causes:
    • Software bugs or incompatibilities.
    • Corrupted installation files.
  • Solutions:
    • Restart CorpuScript and attempt the operation again.
    • Reinstall the application to ensure all files are intact.
    • Check for updates that may address known issues.
    • Contact Support with detailed error logs for assistance.

6. Additional Features

CorpuScript is packed with additional features designed to enhance your preprocessing workflow and provide deeper insights into your corpus.

6.1. Detailed Summary Reporting

After processing your files, CorpuScript generates a comprehensive summary report that provides invaluable insights into your corpus:

  • Word Frequency Distributions:
    • Lists the most common words and their occurrence counts.
  • Sentence and Token Counts:
    • Provides statistics on the number of sentences and tokens processed.
  • Type-Token Ratio Analysis:
    • Measures lexical diversity by comparing the number of unique words to the total number of words.
  • Corpus Size Statistics:
    • Shows the size of your corpus before and after preprocessing.
  • Applied Preprocessing Parameters Summary:
    • Lists all the preprocessing options and parameters that were applied.
  • Processing Time and Performance Metrics:
    • Details the total time taken to process the corpus and other performance-related information.

Accessing the Summary Report:

  1. Navigate to the "Summary Report" tab located alongside the text tabs.
  2. Review the generated statistics and analyses to gain insights into your corpus.

Exporting the Report:

  1. Click the "Export Report" button within the "Summary Report" tab.
  2. Choose your preferred format (.txt or .csv) and select the destination folder.
  3. Click "Save" to export the report for future reference or analysis.
